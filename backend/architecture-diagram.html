Port 8080
user-service
Authentication, JWT issuance, OAuth2 via GitHub, user profile management, admin user operations.
REST Endpoints
POST
/api/v1/auth/signup
Register email/password user. Returns JWT cookie.
POST
/api/v1/auth/login
Authenticate, issue HTTP-only JWT cookie.
POST
/api/v1/auth/logout
Clears JWT cookie (Max-Age=0).
GET
/api/v1/users/me
Returns UserProfileDto (id, name, email, role, hasPassword).
PATCH
/api/v1/users/me/name
Update display name.
PATCH
/api/v1/users/password
Change password (email accounts only). Verifies current password.
GET
/api/v1/admin/users
List all users (ADMIN only).
PATCH
/api/v1/admin/users/{id}/role
Change user role. Self-demotion guard.
PATCH
/api/v1/admin/users/{id}/toggle-active
Activate/deactivate account.
GET
/login/oauth2/code/github
GitHub OAuth2 callback → JWT + redirect.
Kafka Events Published
user.events.v1 USER_REGISTERED on signup
user.events.v1 USER_LOGGED_IN on login (any method)
Storage
PostgreSQL users table: id, name, email, password, role, isActive
Security
JWT HS256, HTTP-only cookie, 24h expiry
OAuth2 GitHub provider, {OAUTH} sentinel password
Key Patterns
getOrCreateOAuthUser() — idempotent user creation on first OAuth login
hasPassword flag on UserProfileDto — frontend uses to show/hide change-password form
Self-demotion guard — admin cannot remove their own ROLE_ADMIN
OTEL
OTEL All HTTP spans + Kafka producer spans exported to collector:4318

Port 8080 · Saga Orchestrator
order-service
Creates and tracks orders. Orchestrates the payment saga via Kafka events. Handles cancellation with compensation.
REST Endpoints
POST
/api/v1/orders
Create order from cart. Persists PENDING, publishes ORDER_INITIATED.
GET
/api/v1/orders
User's order history. Polled every 3s while any order is PENDING.
GET
/api/v1/orders/my-latest
Latest order for payment redirect flow.
GET
/api/v1/orders/{id}
Single order detail.
PATCH
/api/v1/orders/{id}/cancel
Cancel order. Publishes ORDER_CANCELLED compensation event.
Kafka Topics
orders.events.v1 Publishes: ORDER_INITIATED, PAYMENT_SUCCESS, ORDER_CANCELLED, ORDER_SHIPPED, ORDER_DELIVERED
payments.events.v1 Consumes: PAYMENT_SUCCESS, PAYMENT_FAILED → updates order status
Consumer Group
order-service-group-v2 · auto-offset-reset: earliest
Key Bugs Fixed
@Header required=true blocked consumer on missing header → changed to required=false with payload fallback
Consumer group ID mismatch (v1 vs v2) → messages skipped
Kafka publish before DB commit → afterCommit hook via syncAndSend()
No idempotency → duplicate events on redelivery → added no-op guard
Storage
PostgreSQL orders table: id, customerId, status, items (JSON), total
OTEL
OTEL Kafka traceparent propagation — single trace spans order + payment + notification
Port 8080 · Saga Participant
payment-service
Processes payments via Stripe (or mock in dev). Participates in the order saga. Publishes outcome events.
REST Endpoints
POST
/api/v1/payments/initiate
Create Stripe PaymentIntent. Returns clientSecret for frontend.
GET
/api/v1/payments/order/{orderId}
Get payment record for an order.
POST
/api/v1/payments/webhook
Stripe webhook (payment_intent.succeeded / failed).
Kafka Topics
orders.events.v1 Consumes: ORDER_INITIATED → triggers payment processing
payments.events.v1 Publishes: PAYMENT_SUCCESS, PAYMENT_FAILED
Consumer Group
payment-service-group-v6 · auto-offset-reset: earliest
Storage
PostgreSQL payments: id, orderId, stripeRef, status, amount
OTEL
OTEL Propagates traceparent from Kafka into Stripe HTTP spans

Port 8080 · Write Side
review-rating-service
CQRS write side. Accepts new reviews, handles moderation. Publishes ReviewEvents to Kafka for the aggregator to consume.
REST Endpoints
POST
/api/v1/reviews
Submit review (authenticated). Status starts PENDING.
GET
/api/v1/reviews/product/{productId}
List APPROVED reviews for a product.
PATCH
/api/v1/reviews/{id}/approve
Moderator approves review.
PATCH
/api/v1/reviews/{id}/reject
Moderator rejects review.
GET
/api/v1/reviews/pending
All pending reviews (MODERATOR/ADMIN).
Kafka Published
review.events.v1 REVIEW_SUBMITTED, REVIEW_APPROVED, REVIEW_REJECTED
Storage
PostgreSQL reviews: id, productId, userId, rating(1-5), body, status, createdAt
Fixes Applied
Missing SecurityConfig causing 302 redirects instead of 401 JSON
Sub-package import errors → flattened package structure
Missing spring.application.name in application.yaml
Port 8080 · Read Side · CQRS
rating-aggregator-service
CQRS read model. Consumes review events to maintain pre-computed rating aggregates. Serves from Redis cache with 5-minute TTL.
REST Endpoints
GET
/api/v1/ratings/{productId}
Returns { productId, averageRating, reviewCount }. Redis cached.
GET
/api/v1/ratings/top
Top 10 products by average rating.
Kafka Consumed
review.events.v1 REVIEW_APPROVED → increments count + sum, recomputes average, invalidates Redis
Storage
PostgreSQL rating_aggregates: productId (PK), count, sum, average
Redis Key: ratings:{productId} · TTL: 5 minutes
Consumer Group
rating-aggregator-service-group-v1
Fixes Applied
Wrong primary key type on RatingAggregate entity (surrogate id vs natural productId)
Leftover scaffold files conflicting with new implementation — deleted
Test YAML not picked up — renamed to application.yaml
